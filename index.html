SELECTORS

Selector

Code Example

Description

Resources
Type p{color:green;} Matches type of element in HTML doc. Selects all elements of a given type (I.E: P, img, etc). Least specific selector, easiest to override. Link
Universal *{font-family:Helvetica} Selects all elements of any type. Uses * as universal selector. Link
Class .green{color:green;} Attribute helping select an element.Multiple CSS classes can be applied to one HTML element. Classes can be reused. 2nd most specific selector. Will override Type selector but not ID. Link
id #large{color:green;} Attribute used to give HTML element unique style. Can only be used once and have one value. Most specific selector, will override all. Link
Attribute img[src*='water']{height:50px} Attribute targets HTML elements that already have attributes. Elements of same type (img) can be targeted differently by attribute or attribue value (src='water' or src='summer'). Link
Pseudo-Class p:hover{background-color:lime;} Keyword added to selector which specifies special state of selected elements. I.E: Hovering. Link

line-height p{line-height: 1px} The line-height CSS property sets the height of a line box. It's commonly used to set the distance between lines of text. On block-level elements, it specifies the minimum height of line boxes within the element. On non-replaced inline elements, it specifies the height that is used to calculate line box height. Values can be em, px, number or %. Link

BOX MODEL

Concept

Code Example

Description

Resources
Box Model The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content. Link
Box Model Property: Margin p{margin: 10px 18px;} Margins are used to create space around elements, outside defined borders. With CSS, you have full control over the margins. There are properties for setting the margin for each side of an element (top, right, bottom, and left). Link
Box Model Property: Border p{border: 10px normal blue;} The CSS border properties allow you to specify the style, width, and color of an element's border. Link
Box Model Property: Padding p{padding: 3px 40px 10px 18px} The CSS padding properties are used to generate space around an element's content, inside of any defined borders.With CSS, you have full control over the padding. There are properties for setting the padding for each side of an element (top, right, bottom, and left). Link
Box Model: Width & Height h1{height: 40px width: 30px;} Property defines the color of the text. Link
Box Model: Margin Collapse Top and bottom margins of elements are sometimes collapsed into a single margin that is equal to the largest of the two margins.This does not happen on left and right margins! Only top and bottom margins! Link
Box Model: Horizontal Margins Unlike vertical margins, horizontal margins add so the total space between borders of adjacent elements is equal to the sum of the right margin of one element and left or the adjacent. Link
Box Model: Margin Auto Horizontally centers an element inside of parent content area if it has a width. Needs width. Margin: 0 auto; Link
Box Model: Overflow p{Overflow: scroll} This flag will override any style. Difficult to override so don't use unless you need to. Link
Box Model: Visibility p{Visibility:Hidden} Property helps hide elements from view. Common values include hidden, visible and collapse. Difference between display:none and visibility:hidden is that the latter is not removed entirely from the code/page. Link
Box Model: Default box sizing p{box-sizing: content-box} Box sizing property controls the box model used by the browser. Default value is content-box. Affected by border or padding changes. Link
Box Model: Border box p{box-sizing: border-box} Border box value makes sure dimensions of elements remain the same regardless of border width and padding. Height and weight are fixed and padding and border are inside of the box. Link

DISPLAY AND POSITIONING

Concept

Code Example

Description

Resources
HTML Flow When browsers render elements in the same order as they exist in the doc. CSS can style HTML elements but can also change how browser positions elements.
5 Positional Properties: Position p{posiion: value} The position property specifies the type of positioning method used for an element (static, relative, absolute, fixed, or sticky). Link
5 Positional Properties: Position: static p{posiion: static} Default value. Elements render in order, as they appear in the document flow. Link
5 Positional Properties: Position: absolute p{posiion: absolute} The element is positioned relative to its first positioned (not static) ancestor element. The element will still move with the rest of the document when the page is scrolled. Removed from the flow of the document. Link
5 Positional Properties: Position: fixed p{posiion: fixed} The element is positioned relative to the browser window. The element will remain in view no matter what. Removed fromt he flow of the document. Link
5 Positional Properties: Position: relative p{posiion: relative} The element is positioned relative to its normal position, so "left:20px" adds 20 pixels to the element's LEFT position. Link
5 Positional Properties: Position: sticky p{posiion: sticky} The element is positioned based on the user's scroll position. A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position:fixed). Link
5 Positional Properties: Position: initial p{posiion: initial} Sets the property to the default value. Link
5 Positional Properties: Position: inherit p{posiion: inherit} Inherits this property from its parent element. Read about inherit. Link
5 Positional Properties: Display p{display: value} The display property allows you to control how an element flows vertically and horizontally in a document. Link
5 Positional Properties: Display: inline p{display: inline} Displays an element as an inline element (like span).Inline elements take up as little space as possible, and they cannot have manually adjusted width or height. Displayed on same line as other elements. Link
5 Positional Properties: Display: block p{display: block} Displays an element as a block element (like p). It starts on a new line, and takes up the whole width. Link
5 Positional Properties: z-index p{z-index: 30} The z-index of an element specifies how far back or how far forward an element appears on the page when it overlaps other elements.z-index only works on positioned elements (position: absolute, position: relative, position: fixed, or position: sticky. Link
5 Positional Properties: float p{float: left} The float CSS property places an element on the left or right side of its container, allowing text and inline elements to wrap around it. The element is removed from the normal flow of the page, though still remaining a part of the flow (in contrast to absolute positioning). Link
5 Positional Properties: clear p{clear: left} The clear property controls the flow next to floated elements. Elements are pushed below left, right or both left and right floated elements. Link
